# Git

# 基本

  • git status以查看在你上次提交之后是否有修改。

# 分支切换

# 切换分支
git checkout TARGET_BRANCE
# 新建分支并切换分支
git checkout -b TARGET_BRANCE
# 合并分支提交到仓库
git merge TARGET_BRANCE
git push origin CURRENT_BRANCE

# 回滚

  • 未操作git add之前 git checkout 命令
# 单一文件
git checkout -- FILE_PATH
# 撤销所有
git checkout -- .
  • 操作git add之后用 git reset 命令
# 单一文件
git reset HEAD FILE_PATH
# 撤销所有
git reset HEAD
  • 操作git commit之后 用 git revert 命令
# 查看提交记录
git log
# commit 1733527b914c7dec93a4fsdw393999f18ea38225(这个是commitID) (origin/dev)
# Author: ------------
# Date:   ------------
git  revert 1733527b914c7dec93a4fsdw393999f18ea38225

注意的是 revert 奇数次生效，偶数次又回到之前的修改状态。比如一个文件内容是 a，那么修改为 ab，revert 后文件变成了 a，如果在 revert 后文件又还原成 ab 了。

  • git reset --hard commit_id
  • --hard # 强制将缓存区和工作目录都同步到你指定的提交

# 删除文件并推送到远程仓库

git rm -r --cached YOUR_FILE ##--cached不会把本地的YOUR_FILE删除
git commit -m 'YOUR_COMMIT_MESSAGE'
git push -u origin master
